Why simplified

This is Phase 1. The marker expects proposed GUI/wireframes and design reasoning, not a production-looking analytics product.

What changed

Reduced polish, simpler layout boxes, fewer fake metrics, clearer report/PDF export, clearer role permissions.

What stays

The core workflow remains: Handbook data, mapping, assessment setup, CSV upload, calculation, dashboard, local LLM summary, report export.

System workflow for the presentation

Use this simplified flow when explaining the UI and architecture.

1. Handbook data

Scrape unit and LO information, with manual fallback.

2. Setup

Map LO-PLO links and tag assessments to LOs.

3. Upload grades

Validate CSV rows before committing data.

4. Calculate

Distribute weights and compute LO attainment.

5. Report

View dashboard, generate local LLM summary, export PDF.
